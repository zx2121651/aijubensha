# 剧本杀 App 前端开发计划书：多级页面与交互补全

本文档旨在针对目前项目中 React 前端与 Flutter 移动端框架在多级页面、次级视图（尤其是符合 Soul App 风格的沉浸交互与 BottomSheet 抽屉模式）的严重缺失，制定一份分阶段、可执行的详细开发计划书。

## 阶段一：React 前端核心体验补全与深层交互优化 (2-3周)

本阶段优先完善已经具备一定基础的 Web 前端，打通核心游戏与社交环节的二级/三级体验闭环。

### 1. 社区广场与社交链补全
*   **1.1 帖子多级互动增强 (PostDetail)**
    *   **开发任务:** 重构 `PostDetail.tsx`。
    *   **细分步骤:**
        *   实现评论的“楼中楼”展示组件（回复他人的回复）。
        *   设计并实现点击任意评论触发“底部操作抽屉 (BottomSheet)”，提供：回复、点赞、举报、复制内容等操作。
*   **1.2 沉浸式个人主页与社交图谱**
    *   **开发任务:** 替换现有的简陋 `PublicProfileModal.tsx`，建立类似 Soul 的多级个人主页体系。
    *   **细分步骤:**
        *   新增 `UserProfile.tsx` 全屏/半屏路由页，展示该用户的剧本偏好、历史动态、公开成就库。
        *   新增“粉丝/关注/互相关注”列表页。
        *   新增“私信页 (ChatRoom)”入口，支持文字、简单表情包发送。

### 2. 剧本与商业化链路闭环
*   **2.1 剧本详情的深层拓展 (ScriptDetail)**
    *   **开发任务:** 完善选本时的用户体验。
    *   **细分步骤:**
        *   将评价模块抽离，增加 `ScriptReviews.tsx` 二级页面，支持好/中/差评筛选过滤。
        *   优化选角展示，点击角色卡片弹出 `CharacterDetailModal` (包含人物技能、前置小剧场)。
*   **2.2 无缝钱包与支付抽屉**
    *   **开发任务:** 避免因为余额不足导致的全屏跳出。
    *   **细分步骤:**
        *   封装全局 `WalletRechargeDrawer.tsx`。在购买剧本或打赏时若金币不足，直接自下而上滑出充值面板。
        *   完善 `Wallet.tsx` 的子页面：交易流水列表 (`TransactionHistory`) 及其多级明细页。

### 3. 游戏内极致体验打磨 (Game)
*   **3.1 线索与道具的深度交互**
    *   **开发任务:** 强化游戏阶段的代入感。
    *   **细分步骤:**
        *   实现线索放大器组件：支持双指缩放查看线索高清图。
        *   新增线索操作面板 (BottomSheet)：点击线索后，可选择“公开”、“私下塞给某人”、“尝试组合”。
*   **3.2 复盘与结算**
    *   **开发任务:** 游戏结束的荣誉展示。
    *   **细分步骤:**
        *   新增 `ResultDetail.tsx` 页面，包含：真凶揭秘动效、玩家得分榜、MVP颁奖动画，并提供一键分享至社区的入口。

## 阶段二：React 前端 B 端与 DM 端工作台建设 (2周)

补齐 API 设计中存在但前端空白的角色工作台。

### 1. DM (主持人) 专属控场中心
*   **开发任务:** 提供给 DM 上帝视角的管理端。
*   **细分步骤:**
    *   新增 `DmConsole.tsx`。
    *   实现左侧玩家列表卡片，支持一键“全员禁言/解禁”、“踢人”、“标记死亡”。
    *   实现右侧剧本进度树，DM 可点击节点强制触发下一阶段。
    *   实现线索库管理面板，DM 可手动拖拽线索分发给特定玩家。

### 2. 创作者编辑器平台 (Creator Editor) 骨架搭建
*   **开发任务:** 为第三方作者提供内容生产工具。
*   **细分步骤:**
    *   搭建编辑器外框布局 (侧边栏菜单 + 主画布区)。
    *   实现 `ProjectList` 项目大厅。
    *   实现基础的 `ScriptNodeGraph` (剧本节点图编辑器) 原型，采用类似 React Flow 的连线交互。

---

## 阶段三：Flutter 移动端架构升级与多级页面复刻 (3-4周)

在 Web 端逻辑基本验证通过后，将完整的深层交互体验 1:1 复刻至 Flutter 原生端，重点攻克交互动画和 BottomSheet。

### 1. 基础工程结构优化
*   **细分步骤:**
    *   引入 `go_router` 完善深层嵌套路由和转场动画配置。
    *   封装统一的全局 BottomSheet 工具类，确保弹出层风格一致、自带高斯模糊背景。

### 2. 社区与个人中心 (对应 React 阶段一.1)
*   **细分步骤:**
    *   实现 `post_detail_screen.dart`，支持多图横向滑动浏览、嵌套评论列表。
    *   实现发帖页 `create_post_bottom_sheet.dart` (采用半屏拉起)。
    *   实现多级个人主页 `user_profile_screen.dart` 及粉丝/关注列表子页。
    *   实现消息详情页 `chat_screen.dart`，并接入 WebSocket 消息推送。

### 3. 剧本库与商业化 (对应 React 阶段一.2)
*   **细分步骤:**
    *   完整实现 `script_detail_screen.dart`。
    *   开发剧本评价列表页 `script_reviews_screen.dart`。
    *   封装原生支付拉起底座，实现 `recharge_bottom_sheet.dart` 和全局钱包流水查询子页。

### 4. 移动端游戏核心体验补完
*   **细分步骤:**
    *   完善 `game_screen.dart` 的多 Tab 切换动画。
    *   实现原生的地图缩放查看组件 (`InteractiveViewer`) 用于搜证小游戏。
    *   集成 `flutter_webrtc`，实现房间内语音连麦悬浮球控件。
    *   开发精美的原生结算动画页 `result_screen.dart`。

---

## 阶段四：测试、优化与发布准备 (1-2周)

*   **联合调试:** 前后端 Socket.io 状态机深度联调，修复高并发/弱网断线重连时的页面状态错乱。
*   **UI/UX 验收:** 检查所有 BottomSheet 弹出时的手势响应、键盘遮挡处理、以及类 Soul App 动效（如匹配雷达）的流畅度。
*   **发布上架:** 打包构建前端静态资源部署，Flutter 端打包 APK 和 IPA 进入内测。
